## 2026-03-31 - Source 1 Deep Dive

Source:
- Grillini, Thomas, George, "Comparison of Imaging Models for Spectral Unmixing
  in Oil Painting", Sensors 2021.
- Article: https://www.mdpi.com/1424-8220/21/7/2471
- Accessible PDF mirror used for inspection: https://jbthomas.org/Journals/2021bSensors.pdf

### Confirmed facts from the paper

- The mockup set uses seven real pigments from Kremer:
  - Kremer White (`W`)
  - Ultramarine Blue (`B`)
  - Naples Yellow (`Y`)
  - Carmine (`C`)
  - Vermilion (`V`)
  - Viridian Green (`G`)
  - Gold Ochre DD (`O`)
- Support is pre-primed linen canvas with an added acrylic gesso layer.
- Pigments were weighed as powders on a precision scale with `0.005 g`
  sensitivity before being bound in linseed oil.
- Concentrations are therefore mass-based, not volume-based.
- Binary mixtures were prepared at `1:1` and `2:1`.
- Ternary mixtures were prepared at `2:1:1`.
- The total number of painted patches is `175`.
- Hyperspectral acquisition used a HySpex VNIR-1800 in the `400-1000 nm`
  range.
- Acquisition started from `186` spectral bands, then the first and last `10`
  were removed, leaving `166` reflectance values per patch.
- Each patch spectrum was obtained by averaging a manually cropped region.

### Ratio notation used by the authors

- `X` = pure pigment, `100%` of one pigment.
- `XY` = binary `1:1`.
- `Xy` = binary `2:1`, where the uppercase pigment is dominant.
- `Xyz` = ternary `2:1:1`, where the first uppercase pigment is dominant.

### Inferred dataset breakdown

This breakdown is inferred from the ratio notation and the reported total of
`175` patches.

- Pure patches: `7`
- Binary patches:
  - `C(7, 2) = 21` unordered pigment pairs
  - each pair has one `1:1` sample and two oriented `2:1` samples
  - total binary patches = `21 * 3 = 63`
- Ternary patches:
  - `C(7, 3) = 35` unordered pigment triplets
  - each triplet has three oriented `2:1:1` samples
  - total ternary patches = `35 * 3 = 105`
- Grand total = `7 + 63 + 105 = 175`

This is a useful sign that the rebuilt dataset should preserve ratio orientation
instead of collapsing mixtures into unordered pigment sets.

### Why this source is useful for us

- The medium is much closer to our target use case than manuscript or heritage
  paper datasets.
- The measurements are physical and concentration-aware.
- The ratio design is simple enough to start with but already richer than a
  `1:1`-only corpus.
- Because concentrations are mass-based, this source gives us a principled
  target for ratio-aware modeling instead of ad hoc percentages.

### Implications for our rebuilt dataset format

The new corpus should not be constrained to `color A + color B = color C`
without ratio metadata. We can still derive binary training rows, but the
canonical format should retain measured composition.

Recommended canonical fields:

- `components`: ordered list of pigments or color endpoints
- `component_ratios`: numeric mass-ratio values in the same order
- `ratio_basis`: `mass`
- `mixture_arity`: `1`, `2`, or `3`
- `measured_reflectance`: spectral vector
- `wavelengths_nm`: wavelength grid
- `support`
- `binder`
- `acquisition_setup`
- `source_id`
- `source_license`

Recommended derived views:

- Binary residual-correction slice:
  - keep only arity-2 samples
  - preserve explicit ratios such as `[1, 1]`, `[2, 1]`, `[1, 2]`
- Pair-only evaluation slice:
  - group by unordered pigment pair, but never discard the directional ratio
    information

### Current recommendation

- Build the ingestion pipeline around a ratio-aware canonical schema.
- Start modeling with the binary subset first.
- Keep ternary samples in the source-of-truth dataset from day one, even if the
  first residual experiments ignore them.

### Access notes

- The article text and PDF are accessible.
- The paper states that the study data are in the supplementary material.
- Automated fetches against the MDPI supplement endpoint returned `403`.
- A guessed PMC mirror endpoint for the supplementary zip resolves to a

Practical implication:

- We should expect one of these to be necessary before ingestion:
  - manual browser download of the supplementary zip
  - direct author contact if the supplement is incomplete or blocked

### Decision after this pass

- Treat this source as the right first target.
- Treat ratio-aware storage as a requirement, not a nice-to-have.
- Do not carry over the current synthetic/derived dataset as the base for the
  new residual-correction corpus.

Derivation choices:

- Inputs use measured pure swatches as the source for paint `A` and paint `B`
- Targets use the measured binary swatches as `C`
- The binary row format remains simple:
  - input colors
  - integer `parts`
  - measured target color
  - provenance back to the raw measured sample
- The old curated dataset is not reused or merged at this stage

Color conversion note:

- The measured source reflectance is sampled at `186` wavelengths from
  `405.37-995.83 nm`
- The runtime reflectance-to-color converter expects `38` samples on a
  `380-750 nm` grid at `10 nm` steps
- We therefore resample the measured reflectance onto the runtime grid using
  linear interpolation with edge clamping
- Because the measured source starts at `405.37 nm`, the `380-400 nm` runtime
  samples are clamped to the first measured reflectance value

Reason for this choice:

- it keeps the derived colors on the same conversion path used by the runtime
- it lets us postpone any deeper decision about alternate colorimetric
  reconstruction until after more measured sources are ingested

Main findings:

- The reflectance-to-color path itself looks internally sane.
- Flat-reflectance sanity checks behave as expected:
  - `1.00 -> #FFFFFF`
  - `0.75 -> #E1E1E1`
  - `0.55 -> #C4C4C4`
  - `0.25 -> #898989`
- The missing `380-400 nm` region in the measured source has negligible impact
  on the final colors for this dataset.
- Converting on the measured wavelength grid and converting after resampling to
  the runtime grid produces almost identical results for the pure swatches.

Interpretation:

- The unexpectedly dark and muted source-1 colors are mostly coming from the
  measurements themselves, not from an obvious bug in our conversion path.
- The pure `W` swatch is especially informative:
  - min reflectance `0.108`
  - max reflectance `0.488`
  - mean reflectance `0.251`
- That means source 1 simply does not contain a bright near-perfect white
  endpoint. Its pure pigments are measured as relatively dark, muted paint
  patches.

Decision:

- Keep source 1 as valid measured data.
- Treat the derived colors as faithful to this source, but not necessarily
  representative of intuitive tube-paint swatches.
- Use the corrected direct-on-measured-grid conversion path for the derived
  binary ground-truth dataset.

  to collapse those variables away during ingestion.

## 2026-03-31 - Sensors 2021 Color Discrepancy Investigation

Question investigated:

- why do several pure Sensors 2021 swatches look implausible in the gallery,
  for example Naples Yellow appearing as `#4B5357` rather than yellow?

Created:

- `/Users/tarikrahmatallah/programming/spectralnn-paint-mixer/tools/dataset-generator/analyze_sensors_2021_pure_swatches.py`
- `/Users/tarikrahmatallah/programming/spectralnn-paint-mixer/artifacts/measured/sensors-2021-v1/diagnostics/pure-swatch-diagnostics.txt`
- `/Users/tarikrahmatallah/programming/spectralnn-paint-mixer/artifacts/measured/sensors-2021-v1/diagnostics/pure-swatch-reflectance.png`

What was checked:

- importer bookkeeping:
  - concentration and reflectance sample headers match exactly
  - inferred pigment order matches the published code order
  - pure swatch IDs map consistently to the expected single-pigment columns
- color conversion:
  - flat-reflectance sanity tests still produce neutral grays and white
  - trimming the first and last `10` bands, as described in the paper, barely
    changes the derived pure colors
  - the same conversion path yields more plausible colors for other measured
    sources such as Bath 2016
- source spectra:
  - the pure Sensors 2021 swatches are already unusually dark and compressed in
    the visible band before any RGB conversion takes place
  - even the supposed white swatch has a visible-range average reflectance of
    only about `0.121`
  - Naples Yellow has a visible-range average reflectance of about `0.092` with
    no strong yellow-like lift around the red-green region

Conclusion:

- no clear bug was found in the dataset construction pipeline for source 1
- the implausible gallery colors appear to be inherited from the source spectra
  themselves, or from source-side preprocessing/calibration, rather than from a
  label-mapping or final color-conversion mistake in our code

Practical implication:

- treat Sensors 2021 as a low-confidence measured source for color appearance
- keep it for now because it still provides real physical mixtures and useful
  ratio structure
- do not weight it equally with the better-looking measured or screenshot-based
  sources in later experiments unless we discover a better normalization path
